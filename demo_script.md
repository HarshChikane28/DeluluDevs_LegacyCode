## Demo Flow (3–5 minutes)

### 1. Set the Scene (30 seconds)
> "India's IT sector manages billions of lines of legacy COBOL and Java code. Rewriting it manually is slow and expensive. AI can help — but naive approaches fail because LLMs hallucinate when the code context is too large or too noisy. Our tool solves this with a technique we call **Context Optimization**."

### 2. Show the App (30 seconds)
- Open the live URL
- Point out the clean UI — toggle between **GitHub URL** and **File Upload** modes
- Drag in the sample Java file from `sample_project/` OR paste a GitHub repo URL

### 3. Start Translation (1 minute)
- Click **"Start Modernization"**
- Point to the **Progress Stepper** — each phase is tracked: Parse → Graph → Translate → Validate → Package
- As parsing runs, watch the **Dependency Graph** build node by node
- Say: *"Every function is extracted and connected by who calls who"*

### 4. The Key Technique — Context Optimization (1 minute)
- As translation runs, point to the **Translation Log**
- Say: *"Leaf functions — the ones with no dependencies — are translated first"*
- Say: *"When a caller is translated, its already-translated callees are included in the prompt. The LLM always has full context, but never more than it needs"*
- Say: *"Before sending to the LLM, we strip boilerplate comments and dead code, but keep reasoning comments — the ones that explain why the code does what it does"*
- Watch a node turn green on the graph as each function completes

### 5. Validation & Output (30 seconds)
- Show the **compilation validation** step — translated Python is actually run through the compiler
- If the fix loop triggers, say: *"If it fails to compile, we automatically send the error back to the LLM and retry — up to 3 times"*
- When the job completes, show the **Download Panel** with stats: functions translated, success rate, tokens used
- Download the ZIP and open it to show real Python output

### 6. Technical Highlights (30 seconds)
- *"We use Groq's free Llama 3.3 70B as the primary model — with Gemini 2.0 Flash as an automatic fallback if Groq rate-limits"*
- *"The Python requirements.txt is auto-generated by analyzing the translated code's imports"*
- *"Everything — graph updates, translation progress, errors — streams live via WebSocket"*
